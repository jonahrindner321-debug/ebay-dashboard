Security Policy

Security Policy

Seller OS is designed around least-privilege, read-only platform access, encrypted credential storage, and store/client data isolation.

Last updated: April 25, 2026

Security Controls

  • OAuth authorization: Connected seller accounts authorize access through platform-approved authorization flows where supported.
  • Read-only access: Seller OS requests read/reporting scopes and avoids write, delete, fulfillment-control, or account-control permissions unless explicitly required and reviewed in the future.
  • Encrypted tokens: OAuth tokens are encrypted before being stored.
  • Secret management: API keys and secrets are stored in production environment variables and are not committed to the codebase.
  • HTTPS: The production dashboard and API endpoints are served over HTTPS.
  • Data isolation: Connected platform accounts are mapped to the correct store/client so account data is not intentionally mixed across clients.
  • Restricted administration: Production access is limited to authorized administrators and technical operators.

Incident Response

Suspected security issues are reviewed by authorized administrators. If needed, Seller OS can rotate credentials, revoke tokens, disable integrations, disconnect seller accounts, and notify affected sellers or clients according to applicable obligations.

Vulnerability Management

Application changes are reviewed before deployment. Dependencies, serverless functions, and access controls are updated as the product matures.
